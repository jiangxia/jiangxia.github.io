tabNavigator 实现底部导航菜单的流程

  1. 通过npm安装这个插件
  2. 通过“import+组件名字”将组件导入进项目
  3. 通过tabNavigator的TabNavigator来初始化每个选项卡
  4. 细节
    • 通过tabNavigator的selected来标识item被选中的状态
    • selectedTitleStyle来指定标题被选中时的样式变化
    • tintColor为图标进行着色
    • 通过selectedTab值的改变，来切换到不同的页面

什么是Navigator组件呢？

  1. Navigator是一个导航组件，
  2. Navigator可以进入下一个界面，或者返回到上一个界面，
  3. Navigator可以传递数据给下一个界面，返回数据给上一个界面

使用react-navigation替代navigator

具体使用：

  1. 通过react-navigation的initialRoute属性制定react-navigation默认显示的界面
  2. 通过react-navigation来设置跳转的配置，将navigator传给下一级页面，下一级页面才可以调用navigator的pop、push等方法，
  3. react-navigation显示的第一个页面，可以通过push方法来跳转到下一级页面，此时可以通过component指定跳转页面的组件名称，可以通过params给下一级页面传递参数，在params里，可以设置回调函数，然后在下一级页面中，通过this.props.onCallBack的形式向上一级页面传递参数。

NavigationBar是什么？

头部导航栏

NavigationBar的基本组成有什么？

左侧按钮、右侧按钮、标题

设置NavigationBar警告：Invalid props.style key ‘barStyle’ supplied to ‘View,

NavigationBar中有statusBar时，在给NavigationBar设置背景颜色时，也要设置statusBar的背景颜色

设置NavigationBar的流程？

  1. 新建一个JS文件，也就是新建一个NavigationBar组件。该组件的作用是自定义NavigationBar，有返回键、右侧按钮、标题，状态栏
  2. 为了使用者在调用组件时符合要求，需要给组件设置类型检查（propTypes）。对于一些没必须的属性，可以设置默认值。
  3. 在render方法中返回组件需要渲染的部分。
  4. 设置样式，需要分别对安卓跟IOS进行设置。

加载网络图片为什么不显示？

网络图片需要指定宽高，本地图片可以不需要

如何使用ListView?

  1. 导入ListView
  2. 在组件的构造方法中为ListView创建一个数据源ds，然后在state中设置一个数据源，通过克隆方法为DataSource设置一个初始值
  3. 然后在render方法中渲染ListView，并为ListView指定属性
    • 通过dataSource关联在state中设置的dataSource
    • 通过renderRow为ListView的每一行指定显示效果
    • 通过renderSeparator为行与行之间显示分割线
    • 通过renderFooter显示页脚
    • 通过RefreshControl组件为ListView扩展下拉刷新的效果，注意它的refreshing跟onRefresh属性

什么是Fetch？

Fetch API提供了获取资源（比如通过网络）的接口

Fetch与XMLHttpRequest比较，优势是？

  1. XMLHttpRequest不符合职责分离的原则，它将输入输出和事件来跟踪的状态混杂在一个对象里，
  2. Fetch可以与XMLHttpRequest相媲美，同时又提供了更加强大且灵活的特性。Fetch是全局的，不需要额外的导入。

APP的启动引导流程是？

APP启动 => 启动页（读取配置文件） => 状态初始化（从服务器获取配置、更新本地数据状态） => 首页

APP启动的细节是怎样的？

APP启动，会先加载React Native引擎， 然后读取index.js，也就是JS部分的入口，然后执行setup.js，完成相关组件及服务的初始化，之后再进入欢迎页（WelcomePage.js），欢迎页会播放轮播图或者显示广告，然后再进入主页（HomePage.js ）
