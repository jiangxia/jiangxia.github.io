关于加班的思考

发表于 2017-11-06 | 分类于 其他

课程思维导图


Q：加班的原因有哪些？

  1. 主观因素
    • 接触新的业务
    • 接触新的技术领域，技术转型
    • 个人技术追求提升，修整以前觉得不够优雅的解决方案。如页面重构等。
  2. 客观因素
    • 业务的增长过快或开发人手短缺
    • 为解决线上突发情况

Q：减少加班的建议？

  1. 业务导向：在实现功能的时候，多跟业务人员沟通，增加对功能扩展性的把握，方便留出未来的预案。
  2. 技术导向：把代码当成自己的孩子，对每个逻辑都了熟于胸，努力提高自己代码的性能。
  3. 自身建议
    • 勤于总结：不要在同一个坑里摔倒 N 次
    • 保持好奇心：持续学习、拓展技术圈
    • 提高风险意识：做好风险把控，做足测试，减少紧急加班。
